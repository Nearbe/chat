# Версионирование проекта Chat

В проекте используется централизованная система управления версиями инструментов и зависимостей.

## Центральный файл версий
Все версии прописаны в файле:
`Tools/Scripts/Sources/Scripts/Core/Versions.swift`

Этот файл является единственным источником истины для:
- CLI инструментов (XcodeGen, SwiftGen, SwiftLint)
- Зависимостей SPM (Factory, Pulse, SnapshotTesting)
- Настроек платформ (iOS, macOS, Swift)
- Внешних ресурсов документации

## Как это работает

### 1. Загрузка документации (`./scripts download-docs`)
Скрипт использует версии из `Versions.swift` для формирования URL-адресов. Это гарантирует, что локальная документация в папке `Docs/` всегда соответствует версии инструмента или библиотеки, используемой в коде.

Пример: если в `Versions.swift` указан Factory `2.3.0`, документация будет скачана с тега `2.3.0` репозитория Factory.

### 2. Проверка проекта (`./scripts check`)
В процессе проверки (`ProjectChecker`) выполняется верификация:
- Установленных в системе CLI инструментов. Если версия `xcodegen --version` не совпадает с указанной в `Versions.swift`, проверка завершится с ошибкой.
- Файла конфигурации `project.yml`. Проверяется, что версии зависимостей и таргеты в нем синхронизированы с `Versions.swift`.

## Как обновить версию

Если вам нужно обновить инструмент или библиотеку:
1. Измените версию в `Tools/Scripts/Sources/Scripts/Core/Versions.swift`.
2. Если это зависимость SPM или таргет платформы, обновите соответствующие значения в `project.yml`.
3. Запустите `./scripts download-docs`, чтобы обновить локальную документацию.
4. Запустите `./scripts setup` для перегенерации проекта.
5. Запустите `./scripts check` для подтверждения успешного обновления.
