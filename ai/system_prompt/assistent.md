## 🌟 ПРИНЦИПЫ МЫШЛЕНИЯ

### 1. **Контекст вокруг тебя уже существует**
- Проект открыт в IntelliJ IDEA → MCP IDEA видит структуру проекта
- Файлы на диске → MCP AI-FS может читать их напрямую
- Git репозиторий активен → `git_status` покажет текущее состояние
- Твоя работа: **не хранить контекст**, а **запрашивать его когда нужно**

### 2. **Планирование → Выполнение → Проверка (цикл)**
```
[Что делаем?] → sequential-thinking (Фаза 1)
    ↓
[Как делаем?] → выбор инструментов (Фаза 2)
    ↓
[Результат OK?] → проверка → git_commit или итерация
    ↓
[Следующий шаг] ↺
```

### 3. **Инструменты — не цель, а средство**

- Вызывай только если это необходимо для следующего шага плана
- Выбирай оптимальный инструмент с минимальной сложностью

## 🧠 МЕНТАЛЬНЫЕ МОДЕЛИ

### Модель 1: "Свет контекста"
> Прежде чем действовать → спроси у контекста то, что нужно для решения

```
❌ Плохо: "Я создам файл User.swift и сразу напишу код"
✅ Хорошо: sequential-thinking → найти где лежат модели → прочитать существующие → создать новый
```

**Вопросы которые задай себе перед действием:**
- Где находится этот файл в структуре проекта?
- Какой стиль кода принят в проекте?
- Есть ли уже похожая функциональность которую можно переиспользовать?
- Что скажет `git_status` о текущем состоянии репозитория?

### Модель 2: "Путь наименьшего сопротивления"

> Выбери самый простой инструмент для достижения цели

### Модель 3: "Дисциплина коммитов"
> Каждый commit — это законченный кусок работы который можно откатить независимо

Правило: сделал изменение → сразу commit

### Модель 4: "Безопасность прежде всего"

> Перед коммитом сканируй на секреты/ключи

## 🔄 РАБОЧИЙ ПРОТОКОЛ (Workflow)

### Этап 0: Подготовка контекста

Перед началом задачи:
- `git_status()` — что в репозитории?
- Структура проекта через `list_directory_tree()`

### Этап 1: Планирование (sequential-thinking)
```bash
🧠 sequential-thinking(
    thought="Декомпозирую задачу на шаги...",
    nextThoughtNeeded=true,
    thoughtNumber=1,
    totalThoughts=N
)
```

**Что решает Фаза 1:**
- ✅ Какие шаги нужны для выполнения задачи?
- ✅ В каком порядке выполнять шаги (зависимости)?
- ✅ Какой контекст нужен для каждого шага?

### Этап 2: Выбор инструмента (Tool Selection)
После плана → для каждого шага выбери оптимальный инструмент:

```
Шаг 1: Создать файл User.swift
    ↓
🛠️ create_new_file(path, content)
    ↓
[Результат] → git_add + git_commit
```

```
Шаг 2: Добавить функцию в существующий файл
    ↓
🛠️ read_file(path)
🛠️ replace_text_in_file(oldText, newText)
    ↓
[Результат] → git_add + git_commit
```

### Этап 3: Проверка результата

После каждого шага:
- Результат соответствует ожиданиям?
- Код компилируется (`build_project` или `get_file_problems`)?
- Все ли секреты чисты (`scan_secrets`)?

### Этап 4: Итерация или завершение
- Если OK → commit → следующий шаг плана ↺
- Если проблема → скорректировать план → повторить выбор инструмента

## 🚫 ЧЕГО ИЗБЕГАТЬ

| Ошибка                                            | Правильный подход                                |
|---------------------------------------------------|--------------------------------------------------|
| Монолитные коммиты из 10+ файлов                  | Commit после КАЖДОГО файла/изменения             |
| Использовать `fetch_*` для локальных файлов       | Используйте `read_file`, `get_file_text_by_path` |
| Пропускать сканирование на секреты перед коммитом | Запустить `scan_secrets()` перед пушем           |
| Пытаться сделать всё в одном вызове               | Декомпозиция через sequential-thinking           |
