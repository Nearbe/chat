# Реестр игнорируемых системных предупреждений

В этом файле фиксируются все системные предупреждения (Xcode, Simulator, OS), которые были проанализированы и признаны допустимыми для пропуска в процессе автоматической проверки проекта.

## Список исключений

### 1. Metadata extraction skipped (appintentsmetadataprocessor)
- **Текст предупреждения:** `warning: Metadata extraction skipped. No AppIntents.framework dependency found.`
- **Причина возникновения:** Внутренний процесс Xcode `appintentsmetadataprocessor` пытается извлечь метаданные для App Intents даже в проектах, где этот фреймворк не используется или не подключен явно.
- **Что было предпринято для исправления:** 
    - Проверены настройки таргета на предмет лишних зависимостей.
    - Изучена документация Apple по App Intents — предупреждение является побочным эффектом сборки проекта в Xcode 15/16.
- **Обоснование пропуска:** Предупреждение генерируется системной утилитой Xcode, на которую разработчик не имеет прямого влияния. Оно не указывает на ошибку в коде приложения или конфигурации проекта. Исправление требует либо добавления ненужной зависимости `AppIntents`, либо изменения внутренних скриптов Xcode, что нецелесообразно.
- **Паттерн для фильтрации:** `Metadata extraction skipped. No AppIntents.framework dependency found.`

### 2. Ignoring --strip-bitcode (xcodebuild)
- **Текст предупреждения:** `Ignoring --strip-bitcode because --sign was not passed`
- **Причина возникновения:** При сборке или тестировании с параметрами `CODE_SIGNING_ALLOWED=NO` и `CODE_SIGNING_REQUIRED=NO`, Xcode пытается выполнить стриппинг биткода, но это требует подписи кода.
- **Что было предпринято для исправления:** 
    - Проверена возможность отключения стриппинга биткода через флаги `xcodebuild`, однако в новых версиях Xcode биткод считается устаревшим, и это предупреждение часто выводится автоматически при сборке без подписи.
- **Обоснование пропуска:** В CI-окружении (или при локальной проверке без подписи) мы намеренно отключаем подпись для ускорения и упрощения процесса. Это предупреждение не влияет на работоспособность тестов или качество бинарного файла.
- **Паттерн для фильтрации:** `Ignoring --strip-bitcode because --sign was not passed`

## Правила добавления в реестр
Для каждого нового предупреждения, которое необходимо игнорировать:
1. Добавить запись в раздел "Список исключений".
2. Указать точный текст или паттерн.
3. Описать причину, попытки исправления и обоснование.
4. Добавить паттерн в массив `allowedWarnings` в коде скриптов (`Shell.swift`).
