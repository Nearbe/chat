# План миграции скриптов на Swift

## Цели
- Переписать все `.sh` скрипты на Swift.
- Обеспечить доступ к скриптам через IDE (IDEA/Xcode).
- Оптимизировать `check.sh` за счет асинхронного выполнения независимых задач.
- Максимально использовать возможности Swift 6.

## Этапы

### 1. Подготовка инфраструктуры
- Создать Swift Package в директории `Tools/Scripts`. ✓
- Настроить `Package.swift` с необходимыми зависимостями (например, `swift-argument-parser`). ✓
- Добавить `Tools/Scripts` в проект для отображения в IDE. ✓

### 2. Реализация базовых утилит
- Создать общие хелперы для запуска процессов, логирования и работы с файловой системой на Swift. ✓

### 3. Перенос скриптов (исполняемые файлы в SPM)
- `Setup`: Аналог `setup.sh`. ✓
- `SetupAssets`: Аналог `setup_assets.sh`. ✓
- `DownloadDocs`: Аналог `download_all_docs.sh` и всех подскриптов из `scripts/docs/`. ✓
- `Check`: Аналог `check.sh`. Реализовать асинхронный запуск: ✓
    - Группа 1 (Параллельно): `xcodegen`, `swiftlint`, `swiftgen`. ✓
    - Группа 2 (После Группы 1): `xcodebuild build` (Debug). ✓
    - Группа 3 (Параллельно после Группы 2): `xcodebuild test` и `xcodebuild build` (Release). ✓
    - Группа 4 (После успеха всех): `git commit` и `git push`. ✓
- `Ship`: Аналог `ship.sh` (с поддержкой sudo). ✓
- `ConfigureSudo`: Аналог `scripts/configure_sudo.sh`. ✓

### 4. Интеграция с IDE
- Добавить `Tools/Scripts` как локальный пакет в Xcode проект через `project.yml`. ✓

### 5. Завершение и очистка
- Создать тонкие обертки `.sh` в корне (по желанию) или обновить документацию для использования `swift run`. ✓
- Обновить `GUIDELINES.md`. ✓
- Удалить старые `.sh` файлы. ✓

## Ожидаемый результат
- Все скрипты типизированы и легко поддерживаются.
- `check` выполняется быстрее.
- Все скрипты видны и редактируемы в IDE.
