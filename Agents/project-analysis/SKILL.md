# Project Analysis

> **Руководящий принцип**: Объективность — честная оценка состояния проекта без приукрас, чтобы команда видела реальную
> картину.

## Обзор

Координация иерархического анализа проекта с участием всех ролей команды. Процесс проходит 3 волны: сначала исполнители
анализируют свои зоны, затем лиды добавляют экспертную оценку, финально CTO аккумулирует в единый отчёт.

## Активация

Используйте этот навык когда пользователь:

- Просит "провести анализ проекта"
- Говорит "нужен аудит кода"
- Запрашивает "комплексный обзор состояния"
- Хочет "оценить технический долг"

## Права доступа

- **Чтение**: Весь проект
- **Запись**: Корневая директория (для создания отчётов)
- **Коммиты**: Только с разрешения CTO

## Рабочая директория

```
Agents/project-analysis/workspace/
├── cache/              # Кэш предыдущих анализов
├── reports/            # Промежуточные отчёты
└── results/            # Финальные результаты
```

## Процесс анализа

### Волна 1: Исполнители

| Агент                         | Зона анализа                     |
|-------------------------------|----------------------------------|
| Client Developer              | UI компоненты, Views, ViewModels |
| Server Developer              | API, LM Studio, сетевой слой     |
| Designer                      | UI/UX компоненты                 |
| Client QA Lead                | Тесты, покрытие                  |
| Client Security Engineer      | Keychain, уязвимости             |
| Client Accessibility Engineer | VoiceOver, Dynamic Type          |
| Client Performance Engineer   | SwiftData, profiling             |
| Metrics                       | Analytics, метрики               |

### Волна 2: Лиды

| Агент         | Зона                   |
|---------------|------------------------|
| Client Lead   | Client архитектура     |
| Server Lead   | API интеграция         |
| DevOps Lead   | CI/CD                  |
| Head of QA    | Стратегия тестирования |
| Designer Lead | Визуальная концепция   |

### Волна 3: CTO

- Аккумулирует все отчёты
- Создаёт финальный `PROJECT_ANALYSIS.md`
- Приоритизирует проблемы

## Режимы анализа

| Режим           | Описание                                 |
|-----------------|------------------------------------------|
| `--full`        | Полный анализ всей базы                  |
| (по умолч.)     | Инкрементальный — только изменённые зоны |
| `--diff`        | Показать изменения с предыдущего анализа |
| `--roles r1,r2` | Анализ конкретных ролей                  |
| `--review`      | Проверить актуальность прошлых анализов  |

## Взаимодействие

| Агент      | Отношение           |
|------------|---------------------|
| CTO        | Инициирует анализ   |
| Все агенты | Участвуют в анализе |

## Ограничения

- Согласовывайте запуск с CTO
- Документируйте все находки
- Используйте кэширование для скорости
