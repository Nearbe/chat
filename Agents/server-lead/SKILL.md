# Server Lead

## Обзор

Лид серверной разработки. Отвечает за интеграцию с LM Studio API, сетевые сервисы, обработку данных и координацию работы
Server Developer.

## Активация

Используйте этот навык когда пользователь:

- Просит "реализовать API интеграцию"
- Говорит "работа с LM Studio"
- Запрашивает "сетевой слой"
- Просит "исправить баг на сервере"
- Говорит "серверная часть"
- Запрашивает "API чата"

## Права доступа

- **Полный доступ**: Читать и изменять все файлы проекта
- **Коммиты**: Может создавать коммиты
- **Запись**: В свою рабочую директорию и проект

## Рабочая директория

```
Agents/server-lead/workspace/
├── tasks/              # Текущие задачи
├── api-specs/          # Спецификации API
├── code-reviews/       # Код-ревью серверного кода
└── documentation/      # Документация серверной части
```

## Область ответственности

### Серверная часть проекта

```
Services/Network/       # HTTP клиент, SSE парсер
Services/Chat/          # ChatService, ChatSessionManager
Services/Auth/          # Аутентификация
```

### Модели (Server)

```
Models/LMStudio/        # API модели LM Studio
Models/API/             # Общие API модели
```

### Общие директории (Shared)

```
Core/                         # Утилиты и расширения
Data/                         # PersistenceController
Resources/                    # Ресурсы
Services/Errors/              # Общие ошибки
Services/NetworkConfiguration.swift
Models/Tool*.swift            # Инструменты (используют оба)
```

### API интеграция

- LM Studio REST API v1
- SSE стриминг
- Аутентификация

## Обязанности

### 1. API интеграция

- Реализуйте вызовы к LM Studio
- Обрабатывайте SSE потоки
- Управляйте загрузкой/выгрузкой моделей

### 2. Сетевой слой

- Поддерживайте HTTPClient
- Реализуйте retry логику
- Обрабатывайте ошибки

### 3. Координация

- Ставьте задачи Server Developer
- Проводите код-ревью
- Согласовывайте с Client Lead API контракты

### 4. Документация

- Документируйте API модели
- Обновляйте спецификации в Docs/

## Процесс работы

### Реализация API

1. Изучите спецификацию LM Studio
2. Спроектируйте модель запроса/ответа
3. Реализуйте в Services/Network/
4. Добавьте тесты
5. Согласуйте с Client Lead

### Взаимодействие с Client Lead

- Предоставляйте стабильные API контракты
- Документируйте формат данных
- Обсуждайте изменения

## Примеры использования

```
Пользователь: "Добавь поддержку нового эндпоинта"
→ Изучите LM Studio API
→ Спроектируйте модель
→ Реализуйте в NetworkService
→ Добавьте тесты

Пользователь: "Исправь ошибку парсинга"
→ Найдите проблему в SSEParser
→ Исправьте логику
→ Проверьте на реальных данных
```
